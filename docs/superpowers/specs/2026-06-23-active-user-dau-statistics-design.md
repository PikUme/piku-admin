# 활성 사용자/DAU 통계 화면 설계

## 목적

관리자가 로그인 회원의 실제 서비스 이용 규모와 일별 변화를 확인할 수 있도록 통계 상세 페이지의 `방문/DAU` 탭을 `활성 사용자/DAU` 탭으로 개편한다.

추적할 수 없는 방문자 식별 정보에 의존하지 않는다. 신규 방문, 재방문, 재방문율, 가입 전환, 체류 시간은 화면과 데이터 계약에서 모두 제외한다.

## 집계 기준

### 활성 회원

- 로그인 회원만 집계한다.
- 회원이 사용자용 서비스 API를 하루에 한 번 이상 호출하면 해당 날짜의 활성 회원이다.
- 동일 회원이 같은 날 API를 여러 번 호출해도 한 명으로 집계한다.
- 관리자 API, 인증 토큰 갱신, 헬스 체크, 서버 내부 호출은 활동에서 제외한다.
- 날짜 경계는 KST(UTC+9) `00:00:00`부터 `23:59:59.999`까지다.

### DAU

특정 KST 날짜의 활성 회원을 회원 ID 기준으로 중복 제거한 수다.

```text
DAU = 해당 날짜에 사용자용 서비스 API를 1회 이상 호출한 고유 로그인 회원 수
```

### 조회 기간

- `7일`: 최근 완료된 KST 날짜 7개
- `30일`: 최근 완료된 KST 날짜 30개
- `3개월`: 최근 완료된 KST 날짜 90개
- 진행 중인 오늘 데이터는 모든 기간에서 제외한다.
- 화면에는 실제 조회 시작일과 종료일을 표시한다.

예를 들어 조회 시각이 `2026-06-23 KST`라면 7일 범위는 `2026-06-16`부터 `2026-06-22`까지다.

## 화면 구성

### 탭과 제목

- 탭 이름을 `방문/DAU`에서 `활성 사용자/DAU`로 변경한다.
- 차트 제목을 `방문자 추이`에서 `DAU 추이`로 변경한다.
- 상세 표 제목은 `날짜별 DAU 상세`로 변경한다.

### KPI 카드

선택한 기간에 따라 다음 네 개의 카드를 표시한다.

1. **기간 활성 회원 수**
   - 라벨 예시: `7일 활성 회원 수`, `30일 활성 회원 수`, `3개월 활성 회원 수`
   - 선택 기간 전체에서 회원 ID를 중복 제거한 수
   - 같은 회원이 여러 날짜에 활동해도 한 명으로 집계

2. **평균 DAU**
   - 선택 기간의 일별 DAU 합계를 기간 일수로 나눈 값
   - 소수점 첫째 자리에서 반올림하여 정수로 표시

   ```text
   평균 DAU = 일별 DAU 합계 ÷ 기간 일수
   ```

3. **최대 DAU**
   - 선택 기간에서 가장 큰 DAU와 발생 날짜를 함께 표시
   - 예시: `3,105명 · 2026.06.20`
   - 최대값이 여러 날짜에 발생하면 가장 최근 날짜를 표시

4. **누적 활성 회원·일**
   - 선택 기간의 일별 DAU 합계
   - 한 회원이 7일 모두 활동하면 7 회원·일로 계산

   ```text
   누적 활성 회원·일 = Σ 일별 DAU
   ```

기간 활성 회원 수, 평균 DAU, 누적 활성 회원·일에는 직전 동일 길이 기간 대비 증감률을 표시한다. 최대 DAU 카드에는 비교 증감률을 표시하지 않는다.

```text
증감률 = (현재 기간 값 - 직전 기간 값) ÷ 직전 기간 값 × 100
```

- 직전 기간 값이 0이고 현재 값도 0이면 `0.0%`로 표시한다.
- 직전 기간 값이 0이고 현재 값이 0보다 크면 비율 대신 `신규`로 표시한다.

각 KPI 라벨에는 집계 정의를 확인할 수 있는 도움말을 제공한다. 특히 `기간 활성 회원 수`와 `누적 활성 회원·일`의 차이를 명시한다.

### DAU 추이 차트

- 기존 신규 방문/재방문 누적 막대 차트를 단일 DAU 선 차트로 변경한다.
- X축은 KST 날짜, Y축은 DAU 인원이다.
- 7일, 30일, 3개월 모두 일별 값을 사용한다.
- 긴 기간에서는 X축 라벨 일부만 생략하고 데이터 포인트는 생략하지 않는다.
- 포인트에 마우스를 올리거나 키보드로 초점을 이동하면 날짜와 DAU를 표시한다.
- 최대 DAU 날짜의 포인트는 다른 포인트와 구분되도록 강조한다.
- 데이터가 한 건이어도 차트를 정상 표시한다.
- 모든 DAU가 0이어도 축과 0값 포인트를 표시한다.

### 날짜별 DAU 상세 표

다음 열을 최신 날짜부터 내림차순으로 표시한다.

| 열 | 정의 |
| --- | --- |
| 날짜 | KST 집계 날짜, `YYYY.MM.DD` |
| DAU | 해당 날짜의 활성 회원 수 |
| 전일 대비 | `해당 날짜 DAU - 직전 날짜 DAU` |
| 전일 대비율 | `(해당 날짜 DAU - 직전 날짜 DAU) ÷ 직전 날짜 DAU × 100` |

- 첫 조회 날짜도 실제 직전 날짜와 비교한다. 따라서 서버는 조회 시작일 직전 날짜의 DAU를 비교 기준으로 사용해야 한다.
- 직전 날짜 DAU가 0이고 현재 DAU도 0이면 `0명`, `0.0%`로 표시한다.
- 직전 날짜 DAU가 0이고 현재 DAU가 0보다 크면 `+N명`, `신규`로 표시한다.
- 양수는 `+`, 음수는 `-`, 변화 없음은 부호 없이 표시한다.
- 최대 DAU 행은 강조한다.
- 날짜 검색 기능은 유지하며 `YYYY-MM-DD`, `YYYY.MM.DD`, `MM.DD` 입력을 지원한다.

### CSV 다운로드

선택한 기간의 전체 일별 데이터를 현재 정렬 순서대로 내려받는다.

- 파일명: `piku_active_users_dau_{range}_{startDate}_{endDate}.csv`
- 열: `날짜`, `DAU`, `전일 대비`, `전일 대비율`
- 화면 검색어는 CSV 범위를 제한하지 않는다.
- UTF-8 BOM을 포함하여 한글 Excel에서 바로 열 수 있게 한다.

## 데이터 계약

화면은 일별 원천 로그를 직접 집계하지 않는다. 서버가 집계한 요약과 일별 결과를 한 번에 받는다.

```ts
type StatisticsRange = "7d" | "30d" | "3m";

interface ActiveUserDauStatistics {
  range: StatisticsRange;
  timeZone: "Asia/Seoul";
  startDate: string; // YYYY-MM-DD
  endDate: string;   // YYYY-MM-DD
  generatedAt: string;
  summary: {
    uniqueActiveMemberCount: number;
    uniqueActiveMemberChangeRate: number | null;
    averageDau: number;
    averageDauChangeRate: number | null;
    peakDau: number;
    peakDate: string | null;
    activeMemberDays: number;
    activeMemberDaysChangeRate: number | null;
  };
  daily: Array<{
    date: string;
    dau: number;
    changeCount: number;
    changeRate: number | null;
  }>;
}
```

- `changeRate: null`은 직전 값이 0이고 현재 값이 0보다 큰 경우이며 화면에서 `신규`로 표시한다.
- 데이터가 없는 기간의 숫자 요약값은 0이다.
- 데이터가 없는 기간의 `peakDate`는 `null`이다.
- `daily`는 날짜 오름차순으로 반환한다. 차트는 그대로 사용하고 표에서만 내림차순으로 정렬한다.

## 상태 처리

- **로딩:** KPI 카드, 차트, 표 영역에 각각 스켈레톤을 표시한다.
- **빈 데이터:** KPI는 0으로 표시하고 차트와 표에는 `조회 기간의 활성 사용자 데이터가 없습니다.`를 표시한다.
- **오류:** 전체 통계 영역에 오류 안내와 `다시 시도` 버튼을 표시한다.
- **기간 변경:** 선택한 기간의 기존 데이터가 있으면 유지한 채 새 데이터를 로딩하고, 완료 후 교체한다.
- **오래된 응답:** 사용자가 기간을 빠르게 바꿨을 때 마지막으로 선택한 기간의 응답만 화면에 반영한다.

## 제거 대상

`활성 사용자/DAU` 탭에서는 다음 항목을 노출하거나 데이터로 요청하지 않는다.

- 총 방문자
- 신규 방문
- 재방문
- 재방문율
- 가입 전환
- 평균 체류 시간

## 검증 기준

- DAU가 회원 ID 기준 일 단위 중복 제거 값으로 표시된다.
- 모든 날짜와 조회 범위가 KST 기준이다.
- 오늘의 진행 중인 데이터가 포함되지 않는다.
- 기간 활성 회원 수와 누적 활성 회원·일이 서로 다른 정의로 계산된다.
- 평균 DAU, 최대 DAU, 전일 대비 값이 일별 DAU 데이터와 수학적으로 일치한다.
- 7일, 30일, 3개월 변경 시 KPI, 차트, 표, CSV 범위가 함께 변경된다.
- 추적 불가능한 지표가 화면, CSV, 데이터 계약 어디에도 남지 않는다.
- 로딩, 빈 데이터, 오류, 재시도 상태를 키보드와 스크린 리더로 확인할 수 있다.
